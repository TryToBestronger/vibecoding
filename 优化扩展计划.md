# AI热点监控平台 - 优化与功能扩展计划

## 📋 项目概述

本计划旨在对现有的AI热点监控平台进行优化和功能扩展，主要目标：

1. **删除国外信息源**：移除 Reddit、Google、GitHub 的信息获取方式
2. **增加国内信息源**：接入微博、知乎、B站、抖音、小红书等国内平台
3. **账号监控功能**：支持直接监控特定博主/官方账号的动态
4. **无需API Key**：优先使用免费、无需认证的信息获取方式

### 阶段七：账号监控智能识别（预计2小时）

#### 任务 7.1：创建账号识别服务
创建新文件 `server/services/accountDetectionService.js`

**功能：**
```javascript
function detectAccountType(keyword) {
  // 微博账号检测
  if (keyword.startsWith('@') || keyword.includes('微博')) {
    return { type: 'weibo', account: keyword };
  }
  
  // 知乎用户检测
  if (keyword.includes('知乎') || keyword.match(/^[a-z0-9-]{5,}$/)) {
    return { type: 'zhihu', account: keyword };
  }
  
  // B站UP主检测
  if (keyword.includes('B站') || keyword.includes('UP主') || keyword.match(/^UID:\d+$/)) {
    return { type: 'bilibili', account: keyword };
  }
  
  // 公众号检测
  if (keyword.includes('公众号')) {
    return { type: 'wechat', account: keyword };
  }
  
  return { type: 'keyword', keyword: keyword };
}
```

## ⚠️ 六、风险与应对

### 6.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 网页结构变化 | 爬虫失效 | 使用 RSS 作为主要方案，爬虫作为备选 |
| 反爬虫限制 | 数据获取失败 | 降低请求频率，使用多个数据源 |
| API 限流 | 服务不可用 | 实现降级方案，缓存数据 |
| RSSHub 不稳定 | 部分源失败 | 自建 RSSHub 实例 |

### 6.2 合规风险

| 风险 | 应对措施 |
|------|---------|
| 数据爬取合规性 | 仅爬取公开数据，遵守 robots.txt |
| 用户隐私保护 | 不存储用户个人信息 |
| 版权问题 | 仅展示标题和摘要，提供原文链接 |
